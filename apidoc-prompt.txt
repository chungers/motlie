  Organization: Transport implementations first (Local, SSH, Mock with setup +
  characteristics), then all common API built on top. Inconsistencies are
  interspersed at the exact point they arise rather than in a separate section.

  @claude NOTE markers ‚ÄĒ 9 total, each grepable and actionable:

  1. HostHandle::local() hardcodes 10s timeout with no way to configure
  2. TrustFirstUse fail-closed on write failure (intentional but surprising)
  3. is_closed() only on SshTransport, no transport-agnostic equivalent
  4. SSH open_shell() hardcodes 80x24 PTY dimensions
  5. MockTransport HashMap iteration makes multi-pattern matching
  non-deterministic
  6. No way to make MockTransport::exec() return errors
  7. rename() leaves Target with stale name; rename() on pane level errors
  8. TargetSpec::pane() panics instead of returning Result
  9. target.pane(index) from session level resolves via active window
  (non-deterministic)
  10. Two independent timeout knobs (transport vs exec sentinel) with no
  documentation linking them
  11. overlap_lines silently requires >= 2
  12. capture() at session level captures active pane only
  13. history-limit only affects future panes
